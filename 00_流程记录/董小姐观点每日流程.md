# 董小姐观点每日流程

## 目标

每天把董小姐长截图跑成一套可审核、可回看、可复制进 Excel 的输出包。

## 目录约定

- `01_当前资料/`：项目说明、样例 Excel、后续稳定模板。
- `02_OCR工作流/`：OCR 工具本体，只作为临时加工区。
- `03_整理输出/YYYY-MM-DD/`：每天最终保留的输出包。

## 每日输出包

每个日期目录最终只保留 4 类文件：

1. 原始图片：`YYYY-MM-DD_IMG_xxxx.JPG`
2. OCR 正文文本：`YYYY-MM-DD_IMG_xxxx_正文.txt`
3. OCR 正文 Markdown：`YYYY-MM-DD_IMG_xxxx_正文.md`
4. Excel 候选表：`YYYY-MM-DD_Excel候选表.md`

这些文件构成当天可追溯交付物。Excel 暂不直接写入，先用候选表审核。

## 执行步骤

1. 把当天原图放入 `02_OCR工作流/待整理截图/`，统一命名为 `YYYY-MM-DD_IMG_xxxx.JPG`。
2. 单图运行 OCR，输出到 `03_整理输出/YYYY-MM-DD/`：

```bash
cd /Users/a123/Downloads/董小姐/02_OCR工作流
python3 extract_article_text.py \
  --out-dir /Users/a123/Downloads/董小姐/03_整理输出/YYYY-MM-DD \
  --format both \
  --no-combined \
  /Users/a123/Downloads/董小姐/02_OCR工作流/待整理截图/YYYY-MM-DD_IMG_xxxx.JPG
```

3. 抽查 OCR 正文：标题、日期、关键数字、专有名词、正文起止。
4. 生成 `YYYY-MM-DD_Excel候选表.md`，字段按样例 Excel A-K：
   `经济体 / 维度 / 板块 / 议题/指标 / 子项 / 细项 / 原文摘录 / 精简观点 / 态度 / 标签 / 时间`。
5. 把原图从 `02_OCR工作流/待整理截图/` 移到当天输出目录。
6. 清空 OCR 工作区里的临时截图；`02_OCR工作流/` 只保留脚本、说明、命令文件和必要缓存。

## 拆解规则

- 不新增骨架叶子；找不到稳定落点时，在候选表里标 `待确认`。
- 跨主题内容只落主分类，其他主题靠标签保留。
- `原文摘录` 保留完整支撑段落，不使用 `...` 省略；多段证据放在同一单元格时用 `<br>` 分隔。
- `精简观点` 控制在 40-80 字左右。
- `态度` 只用：积极 / 中性 / 谨慎 / 负面。
- `标签` 用 `#` 开头，每条 3-7 个。

## 清理规则

- 原始图片只保留在当天输出包，不留在 OCR 待处理区。
- OCR 临时切图、调试截图、overview 截图用完删除。
- OCR 输出若已进入当天输出包，不再保留在 `02_OCR工作流/整理结果/`。
- 不把原始图片、OCR 文档、候选表、Excel 放进 GitHub。

## 待观察清单

- OCR 标题是否经常缺引号、日期或“解读要点”。
- 正文是否会漏掉开头、结尾或关键段落。
- 每日拆成多少条最合适，避免过粗或过碎。
- 行业主题在现有骨架里是否经常找不到稳定落点。
- 标签是否开始出现同义词分裂，例如 `#AI硬件` / `#算力硬件`。
- 候选表人工审核耗时是否能控制在 10 分钟内。
